# Product brief: Alexsandro Pessoa interactive systems portfolio

**Created:** 2026-07-17  
**Status:** Working contract for implementation and future agents  
**Target:** GitHub Pages at `al4xdev.github.io` or a custom domain

## Single job

Turn a short visit into a clear conclusion: Alexsandro designs and ships ambitious AI systems,
and he can explain the engineering evidence behind them.

The page is not a generic resume, a skill-cloud, or a gallery of disconnected repositories. It is
an interactive map of one body of work: agentic systems, deterministic boundaries, visual AI
workflows, reusable orchestration, and developer ecosystems.

## Audience

- Engineering leaders hiring for GenAI, MLOps, platform, cloud, and automation roles.
- Technical peers evaluating architecture and open-source work.
- Builders arriving from LinkedIn posts, GitHub repositories, or case studies.
- Potential collaborators who need to understand the projects quickly before reading source.

## Positioning

English:

> I build systems where AI judgment meets deterministic control.

Portuguese:

> Eu construo sistemas onde julgamento de IA encontra controle determinístico.

Supporting identity:

- GenAI & MLOps specialist.
- Senior cloud and automation engineer.
- Curious builder turning unusual ideas into working systems.

## Required experience

1. The first screen asks the visitor to choose **Português** or **English**.
2. The language choice is explicit and memorable, not hidden in a header menu.
3. After selection, the page enters an interactive project constellation.
4. Selecting a project updates a focused inspection panel without navigating away.
5. The visitor can switch language later without losing the selected project or scroll position.
6. Keyboard navigation, mobile layout, reduced motion, and clear focus states are mandatory.
7. The GitHub source remains the evidence boundary. The portfolio explains and links; it never
   inflates claims beyond repository evidence.

## Content hierarchy

1. Thesis and professional identity.
2. Interactive system map.
3. Selected project inspection: problem, distinctive decision, evidence, link.
4. Engineering principles extracted from the work.
5. Papers, field notes, and case studies below the project experience.
6. Contact and repository links.

English is the canonical writing language during the first implementation pass. Portuguese remains
available as a product requirement, but final copy editing happens only after the English structure
and claims stabilize.

## Tone

- Serious without becoming corporate.
- Curious without sounding whimsical or vague.
- Specific, evidence-led, and written from the visitor's point of view.
- No inflated AI claims, fake metrics, skill percentages, or generic “passionate developer” copy.

## Scope guard

The first version is a dependency-free static site suitable for GitHub Pages. It establishes the
experience and information architecture. Later increments may add repository media, live GitHub
metadata, deeper case-study routes, a custom domain, and privacy-conscious analytics.
